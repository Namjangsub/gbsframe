WBS KPI 대시보드 사용자 도움말

📊 WBS KPI 대시보드란?

본 대시보드는 프로젝트의 WBS(Work Breakdown Structure) 계획 및 실적 데이터를 시각화하여, 현재 프로젝트의 건강 상태와 위험 요소를 한눈에 파악할 수 있도록 돕는 KPI 분석 도구입니다. 아래 설명을 참고하여 차트를 분석하고 의사결정에 활용해 보십시오.

1. 분포 및 이슈 현황 분석 (Row 1)

📈 계획 기간 분포 (일수) - 정의: WBS 단계별 계획된 소요 일수(Duration)의 총합 분포입니다.
- 해석: 프로젝트 전체 기간 중 설계, 개발, 테스트 등 각 단계에 시간적 자원이 어떻게 배분되었는지 보여주며, 특정 단계의 비중이 과도하게 크거나 작지 않은지 검토할 때 유용합니다.
👥 실적 공수 분포 (Man-Month) - 정의: WBS 단계별로 실제 투입된 인적 공수(MH)의 분포입니다.
- 해석: 실제 투입된 인력 리소스의 집중도를 파악합니다. 계획 기간 분포와 비교했을 때, 특정 단계의 공수 비율이 계획보다 지나치게 높다면 해당 단계에서 병목 현상이 발생했거나 오버워크가 있었음을 의미합니다.
⚠️ 문제 현황 (중요도별) - 정의: WBS 단계별 등록된 이슈의 개수를 심각도/빈도 매트릭스에 따라 누적 표시합니다.
- 구분:
· 🔴 비상빈상(HH): 고긴급/고빈도 핵심 장애 (즉시 조치)
· 🟠 비하빈상(LH): 저긴급/고빈도 만성 이슈
· 🟡 비상빈하(HL): 고긴급/저빈도 돌발 장애
· 🟢 비하빈하(LL): 경미한 개선사항
- 해석: 붉은색 계열(HH, LH)의 비율이 높은 단계는 프로젝트 리스크가 높은 영역이므로 관리자의 즉각적인 개입과 자원 배정이 필요합니다.

2. 종합 KPI 지표 분석 (Row 2 - 게이지)

※ 게이지 바늘이 초록색 영역에 가까울수록 안정적이며, 빨간색 영역에 있을 경우 관리가 시급합니다.

🔵 진행률 (Status) 프로젝트 전체 완료 진척도입니다. 100%를 목표로 계획 대비 현재 진척 상황을 정량적으로 표시합니다.
🟢 준수율 (Schedule) 작업 일정을 계획에 맞춰 준수한 비율입니다. 바늘이 높을수록(100%에 가까울수록) 지연 없이 일정대로 잘 진행되고 있음을 뜻합니다.
🟡 지연율 (Delay) 계획 대비 완료가 지연된 작업의 비율입니다. 반대로 낮을수록 양호하며, 빨간 영역(높음)으로 갈수록 일정 경보 상황입니다.
🟣 일치율 (Alignment) 계획 공수와 실제 투입 공수의 일치율입니다. 100%가 가장 이상적이며, 과소 투입되거나 과다 투입된(150% 근접) 경우 계획성 재검토가 필요합니다.
🔴 리스크 (Risk) 미결 이슈나 미완료된 주요 마일스톤에 따른 리스크 점수입니다. 0%에 가까워야 하며, 바늘이 올라갈수록 프로젝트 실패 가능성이 높아집니다.

3. WBS 단계별 상세 분석 (Row 3 - 막대그래프)

분석 항목 설명 및 의사결정 가이드
실적미완료 현황 (Critical) WBS 상에서는 완료 기한이 도래했으나 실적이 최종 입력되지 않거나 완료 처리가 누락된 항목의 비율입니다. 이 비율이 높은 단계는 진행 현황을 재점검하고 즉시 완료 처리를 독려해야 합니다.
일정 지연 완료 (Late) 작업은 완료되었으나, 당초 계획된 완료일보다 늦게 끝난 항목의 비율입니다. 사후 분석 시 유용하며, 이 비율이 유독 높은 단계는 다음 프로젝트 기획 시 계획 기간을 현실적으로 늘릴 필요가 있습니다.
일정 지연 미완료 (Delay) 현재 기준 완료 기한이 지났음에도 아직 완료되지 않고 진행 중인 상태의 작업 비율입니다. 현재 시점의 병목 구간을 직접적으로 타겟팅하며, 지연 해소를 위한 지원 인력 배치가 필요합니다.
공수 일치 항목 (Align) 계획된 공수 범위 내에서 정확하게 작업을 완료한 항목의 비율입니다. 이 비율이 높을수록 프로젝트 계획의 정밀성과 실행 완성도가 매우 높음을 증명합니다.
문제 발생율 (Risk) 각 WBS 단계별로 등록된 결함/이슈의 가중 비율입니다. 높은 문제 발생율을 기록하는 구간은 향후 QA(품질보증) 단계를 한층 더 강화하여 결함이 운영 환경으로 전이되지 않도록 방지해야 합니다.
